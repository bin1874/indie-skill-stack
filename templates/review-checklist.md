# Review Checklist

Use this before adding a skill to the main list.

## Identity

- [ ] Name is clear.
- [ ] Repo or official page is public.
- [ ] Maintainer or source is visible.
- [ ] License or usage terms are understandable.

## Workflow Fit

- [ ] Helps indie builders validate, ship, launch, operate, or improve real products.
- [ ] Has a specific `Use when`.
- [ ] Has a specific `Best for`.
- [ ] Is not only a generic prompt or toy demo.

## Setup

- [ ] Has direct install, copy, or setup instructions.
- [ ] Can be started within 5 minutes.
- [ ] Does not require one specific agent platform unless that is clearly stated.

## Risk

- [ ] Data access is clear.
- [ ] API keys or credentials are not required by default, or are clearly explained.
- [ ] Shell, file write, network, or account-control behavior is clear.
- [ ] Risk label is assigned: Low / Medium / High / Reject.

## Verdict

```text
Verdict: Include / Hold / Reject
Reason:
Risk note:
Best category:
Last reviewed:
Reviewer:
```
